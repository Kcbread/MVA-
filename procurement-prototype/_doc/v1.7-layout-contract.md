# v1.7 Shared Table And Action Layout Contract

## Summary

This version introduces a shared table and action layout contract to stop repeated button overflow, cross-cell text overlap, and legacy CSS collisions.

## Core Rules

- Every dense table is wrapped by `.table-shell`.
- Every shared table uses `.table-fixed`.
- Columns are classified into:
  - `cell-identity`
  - `cell-spec-summary`
  - `cell-note-summary`
  - `cell-number`
  - `cell-timeline`
  - `cell-action`
- Table action buttons use short labels only:
  - `Add`
  - `Edit`
  - `Remove`
  - `Detail`
  - `Export`
  - `Apply`
  - `Reject`

## DOM Contract

Action cells must follow:

```html
<td class="cell-action">
  <div class="action-stack">
    <button class="layout-action-btn">Edit</button>
    <button class="layout-action-btn">Remove</button>
  </div>
</td>
```

## Identity vs Spec

- Identity fields must remain readable and cannot be hard-truncated.
- Spec fields may be summarized in 2-3 lines, but must expose full content through `title`, `Detail`, or both.

## Covered Areas

- User A add/reuse modal
- User A draft tables
- User A request status
- Manager dashboard tables
- Manager matrix tables
- OM workflow tables
