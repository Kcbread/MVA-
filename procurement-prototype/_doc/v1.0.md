# v1.0 UI Logic Consolidation

## 中文摘要

本版將三個主要角色的第一眼資訊重新對齊業務問題：

- User A `Add / Reuse Item` 主表只放可操作摘要，row action 使用短按鈕 `Add`，完整語意放 tooltip / Detail。
- Manager B `Demand Cost Dashboard` 保持 Excel Dashboard-like 高密度，但 `Selected Phase Department Total` 獨立於 item 明細表，不再混入第一列。
- OM `Submission Dashboard` 改看 pending owner、submitted/received date、days pending、quote status、next action。
- OM 不再把 Budget / PR / PO / Arrived 當主欄；PO 是 Buyer scope，只在 downstream detail 裡追。
- OM Exchange Rate 改為 compact utility，不作為主流程面板。

## English Summary

This version realigns each role's first-view UI with the business question it needs to answer:

- User A `Add / Reuse Item` shows compact actionable summaries. Row action is shortened to `Add`; full meaning is retained in tooltip / Detail.
- Manager B `Demand Cost Dashboard` keeps the Excel Dashboard-like density, but `Selected Phase Department Total` is separated from item detail rows.
- OM `Submission Dashboard` now focuses on pending owner, submitted/received date, days pending, quote status, and next action.
- OM no longer uses Budget / PR / PO / Arrived as primary columns; PO remains Buyer downstream scope.
- OM Exchange Rate is a compact utility instead of a workflow-dominant panel.
