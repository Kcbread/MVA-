# v1.6 Carryover DRI Ownership

## Summary

Carryover is now defined as a DRI-owned inventory and demand adjustment. It is not a new workflow for every role.

## Role Rules

- DRI operates carryover review, apply, and reject.
- Manager B only sees carryover impact on quantity and cost.
- OM Purchasing receives effective qty and does not decide carryover.
- User A does not operate carryover.
- Admin will own future carryover permissions and thresholds.

## Data Rule

Carryover never overwrites original demand. Every line-to-line change creates a ledger event:

- Source Line
- Target Line
- Item
- Phase
- Station / Unit
- Original Qty
- Used Qty
- Carryover Qty
- Effective Qty
- Status
- Confirmed By
- Timestamp
- Reason

## Chain Behavior

Line 2 can become the source for Line 3. The system records this as a second event in the same chain instead of modifying the Line 1 to Line 2 event.

Example:

- Event 1: Line 1 -> Line 2
- Event 2: Line 2 -> Line 3

## UI Placement

- DRI: `Carryover Review`
- Manager B: read-only `Carryover Impact` under Demand Analysis
- OM Purchasing: read-only `Effective Qty from DRI Carryover`

The extension exposes `ProcurementCarryover.effectiveQtyFor(item, phase)` and `carryoverQtyFor(item, phase)` so the core OM export flow can consume effective demand without duplicating carryover logic.
