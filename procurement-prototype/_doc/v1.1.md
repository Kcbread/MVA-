# v1.1 Demand Date + DRI / Project DRI Price Approval Layer

## Scope

This version adds required demand dates and a price-threshold approval layer after OM PAS quote result.

## Business Rules

- OPM / User A must enter `Need Date` before submitting any draft item.
- `Need Date` is stored as both `needDate` and `requiredDeliveryDate` so Manager, OM, and downstream progress views share the same date source.
- OM quote result compares quote unit price with reusable history price.
- Computer items auto-clear when quote price is within history price + 20%.
- MFG items auto-clear when quote price is within history price + 10%.
- No history price, unclassified price category, or `Temporary Budget Request` requires price escalation.
- Auto-cleared rows skip User A confirmation and move directly to OM `Export Package`.
- Escalated rows go through `DRI -> Project DRI`.
- Project DRI approval moves the row to OM `Export Package`.
- Rejection records reason and keeps the row in price review handling.

## UI Changes

- Login adds `DRI` and `Project DRI`.
- Added `Price Review` view:
  - `Pending Price Review`

- Added Admin `Access & Approval Setup`:

- OM quote save classifies rows as `Auto Cleared` or `Price Escalation Required`.
