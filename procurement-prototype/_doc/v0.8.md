# v0.8 Export / Amendment / Dashboard / Compact UX

## 中文摘要

本版收斂四個近期決策：

- OM `Export Package` 改為單一操作，不再分開 `Export Excel` 與 `Export Quote PDF`。
- Export 前改由 OM 選 `Expense / Capex`，系統同步顯示 downstream package 對應：`Expense -> ECS`、`Capex -> CFA`。
- Bidding result 後的需求修改走 amendment trace；若數量改為 0，可在 Manager approval 後標示 `Removed by Amendment`，不再進 Export / Buyer。
- Manager B `Cost Dashboard` 依 Excel `Dashboard` 邏輯，先顯示目前 project/phase/line count 下各 unit 的總量或總金額，再看 item 明細與 Station Matrix。
- User A `Add / Reuse Item` 與 Demand Editor 做 compact UX：Add / Remove 欄固定可見，降低橫向捲動。

## English Summary

This version consolidates four workflow decisions:

- OM `Export Package` becomes one user action instead of separate Excel and quote PDF actions.
- OM selects `Expense / Capex` first; the system displays the downstream package mapping: `Expense -> ECS`, `Capex -> CFA`.
- Post-bidding demand changes use amendment trace. If revised quantity becomes zero, Manager approval marks the source as `Removed by Amendment` and keeps it out of Export / Buyer.
- Manager B `Cost Dashboard` follows the Excel `Dashboard` logic: unit totals by selected project, phase, and line count are shown before item-level details and Station Matrix drilldown.
- User A `Add / Reuse Item` and Demand Editor are made compact with visible action columns and less horizontal scrolling.
