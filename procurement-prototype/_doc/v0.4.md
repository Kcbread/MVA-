# v0.4 - Manager Excel Dashboard + OM Stage Aging

## Summary
- Manager B `Demand Analysis` keeps the two-layer model: first `Cost Dashboard`, then `Station Matrix`.
- `Cost Dashboard` is tightened into an Excel-like table style so it visually aligns with `Station Matrix`.
- OM Purchasing workflow tabs are now `Submission Dashboard / PAS Demand No / PAS Quote Result / Export Package`.
- `Quote Expiry` is no longer a workflow tab. It is monitored inside `Submission Dashboard`.

## Functional Decisions
- OM `Submission Dashboard` tracks when OM received the request and how long the row has stayed in the current OM stage.
- `Received Date` uses `sentToOmAt`, then falls back to `managerApprovedAt`, `decidedAt`, `approvedAt`, then `submittedAt`.
- `Days in Current Stage` is calculated from the current stage start:
  - `PAS Demand No`: received date.
  - `PAS Quote Result`: PAS Demand No recorded date.
  - `Waiting User A`: sent-to-User-A timestamp.
  - `Export Package`: User A confirmation timestamp.
  - `Exported`: completed.
- OM SLA warning threshold remains 7 days.
- `PAS Quote Result` is the only place to enter quote result, files, and quote validity.

## Tests
- `tests/system-contract.test.js` now asserts that OM has no top-level `Quote Expiry` tab or panel.
- The contract also checks that `Submission Dashboard` exposes `Received Date` and `Days in Current Stage`.
- Run `./test.sh` after every large UI or workflow change.

## Known Follow-ups
- Browser smoke still depends on Playwright or Browser runtime availability.
- The next larger refactor should continue splitting large `app.js` render helpers into registered modules.
