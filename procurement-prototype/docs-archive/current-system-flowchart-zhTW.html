FIH Equipment Management Handoff System

現行採購需求系統流程圖

本圖說明目前 prototype 已實作的流程：User A / DRI 提出需求，MFG Coordinator 追蹤 MFG line / phase 是否收集齊，Manager B 審批；IT / OM Buy scope 由 OM Purchasing 先做 PAS Review，PAS 完成後才進 OM 採買 package。

現行系統版 · PAS Gate 已納入
提醒：IT / 資訊網絡 / OM Buy scope 為 PAS 必經流程；BC 預算流程仍屬後續串接規劃。

一、現行主流程

從 MFG 收集、Manager 審批到 OM / Sourcing / Buyer 閉環
User A

Department Item Request

Natural Search / History / Create New Item搜尋既有品項、複製歷史品項，或建立新品項需求。
↓
Request Draft填入 P1.0 / P1.1 / EVT / DVT / PVT / MP 各階段數量。
↓
Submit to Manager B提交給 Manager B 審核；User A 後續在 My Submissions 追蹤狀態。
Manager B

Manager Approval

Review Queue檢查需求、數量、OM Buy Scope、PAS Status、下一步分流。
↓
Demand Tracking / Project Cost View查看殘留、需求風險、成本可信度，作為審批依據。
↓
Approve / Return / RejectReturn / Reject 必填原因；Approved 後進入系統自動 routing。
System Routing

Manager Approved 後自動分流

PAS Required - OM Buy ScopePC、IPC、Laptop、Monitor、Keyboard、Mouse 等 IT / OM 品項先進 PAS Review。
PAS Approved - Send to OMPAS 回覆 project code / budget amount 後，才進 OM Demand Collection。
MFG / Non-IT / Sourcing NeededMFG 或需要供應商詢價的品項，進 Sourcing / Buyer 後段流程。

OM Purchasing 路徑

處理 PAS approved 後的 OM Buy scope / IT recurring bucket，包含可先用 bucket 收集需求的 PC、IPC(A) 等品項。

  • 接收 PAS approved / budget-code-issued demand。
  • 在 Demand Collection 彙整需求。
  • 確認 final spec / model。
  • 內建 IT / OM Buy sourcing 工作，維護 vendor、price、quote date、quote expiry。
  • 上傳 Quote PDF，輸出 OM Excel / PDF package。
  • package 送出或外部接受後，進入 Buyer 外部 PR / PO 追蹤；系統不自行判斷 CFA / ECS。

MFG Coordinator 路徑

負責 MFG 各 line / phase 需求收集完整性，確認 Mainline / Packing / Supporting / Demand Calculation 是否收齊後，形成 package 給 Manager B。

  • 追蹤 MFG collection completeness。
  • 缺資料時退回 line / area owner 補資料。
  • 資料收齊後形成 MFG Demand Package。
  • 必要時配合 RFQ / Sourcing 後段流程。

二、OM / Coordinator / Sourcing / Buyer 的差異

避免角色責任混在一起
角色 主要責任 典型品項 / 情境 目前輸出
OM Purchasing 收集 OM-scope 需求、確認 final spec/model、內建 OM sourcing 工作、維護報價、上傳 Quote PDF、輸出正式 package。 PC、IPC、Laptop、Monitor、Keyboard、Mouse、Barcode、Printer、Network Switch、Software。PC / IPC(A) 可先作為需求 bucket。 OM Purchasing Excel `.xlsx`、Quote PDF package `.pdf`、外部進度與 evidence，送 Buyer 追蹤 PR / PO。
MFG Coordinator 追蹤 MFG line / phase 需求收集完整性，形成 package 給 Manager B。 MFG Mainline、Packing、Supporting、Demand Calculation 收集狀態。 MFG Demand Package、collection status、missing data follow-up。
Sourcing 接收 RFQ 任務，向供應商詢價並回填 quote result、vendor、price、Quote PDF。 MFG / 非 IT 品項，需要供應商報價。 Quote result、vendor / price、Quote PDF、Sourcing RFQ 狀態。
Buyer 追蹤外部系統結果、外部單號、PR No.、PO No. 與佐證附件。 OM package 送出 / 外部接受，或 MFG / Sourcing 完成後的 package。 External Request No.、PR No.、PO No.、Evidence、Completed。

三、目前系統會輸出的檔案格式

給老闆快速理解交付物
.xlsx RFQ Excel

Owner：MFG / Sourcing flow。依 Sourcing Owner 分檔，輸出給 Sourcing 進行詢價。

Email Draft RFQ Email Draft

Owner：MFG / Sourcing flow。產生可複製的 email subject/body，不直接寄信。

.xlsx OM Purchasing Excel Package

Owner：OM Purchasing。包含 `Project budget-summary table` 與 `{Project}-OM-Purchasing-Detail` 兩個 sheet。

.pdf Quote PDF Package

Owner：OM Purchasing / Sourcing。作為報價證據；外部進度與 Buyer PR / PO 需另外留存 evidence。

Audit Trail Activity / History Records

Owner：System。記錄 RFQ dispatch、Sourcing reply、PDF upload、Excel/PDF export、external progress evidence、Buyer PR / PO、revision history。
