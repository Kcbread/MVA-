MVA PROCUREMENT GOVERNANCE

MVA採購資料治理與端到端流程數位化
開發進度

User / Manager MFG Coordinator OM Purchasing Sourcing / Buyer
本階段定位

從分散式郵件作業，升級為可追溯、可稽核、可量化的採購協同平台，建立需求、報價、預算、PR / PO 的資料閉環。

1需求標準化與審批治理

User A / DRI：統一需求入口

以自然語言搜尋、歷史複用與新品項建立，沉澱標準化需求資料。

↓

MFG Coordinator：Line / Phase 完整性收斂

彙整 Mainline / Packing / Supporting 與各階段需求，降低漏提與重複採購。

↓

Manager B：需求合理性與成本風險決策

以 Demand vs Actual Buy、殘留復用與成本信心作為審批依據。

2責任分流與流程閉環

Manager Approved 後形成角色責任鏈

依品項責任歸屬進入 OM Purchasing、Sourcing 或 Buyer 追蹤節點，避免郵件式交接斷點。

↓

OM Purchasing

  • IT / OM Buy 需求收斂
  • PAS 資訊彙整與預算資料承接
  • Final spec/model 確認
  • 報價資料與 Quote PDF 管理
  • 正式 OM Excel + PDF package 輸出

Sourcing / Buyer

  • Sourcing：RFQ 詢價、供應商與報價回覆
  • Buyer：外部系統結果、PR / PO 狀態回填
  • 不重建 CFA / ECS 規則
  • 聚焦外部結果與佐證回收
↓

外部系統進度追蹤

以 Status + Evidence 形成可稽核紀錄，補齊跨系統作業的可追溯缺口。

3外部系統與採購執行追蹤

Package Readiness Control

PAS 資訊、OM Excel、Quote PDF 與送件佐證完成檢核。

↓

External Submission：外部送件結果留痕

記錄 External System、External Request No.，並綁定截圖、貼上結果或附件。

↓

Buyer PR / PO：節點與佐證回填

PR Created、PO Issued、Completed 皆保留 PR / PO No. 與 evidence。

Revision Control：退件與重送治理

退件、修正、重送全程保留版本紀錄，避免責任與資料斷點。

標準化輸出與佐證資料

OM Excel Package預算摘要與採購明細標準格式。
Quote PDF供應商報價與正式附件管理。
RFQ Excel / Email Draft詢價資料標準化輸出與溝通。
External Evidence截圖、貼上文字、email、PDF、Excel、zip / msg。
