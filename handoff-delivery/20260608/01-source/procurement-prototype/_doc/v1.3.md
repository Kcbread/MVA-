# v1.3 Shared Layout Contract Stabilization

## Summary

This version applies the new `frontend-layout-stability` skill to the procurement prototype layout contract. The goal is to stop recurring table text overlap, button misalignment, modal overflow, and clipped amount display.

## What Changed

- Added a final shared layout authority section in `styles.css` for table shells, fixed tables, action cells, text/spec clamps, numeric cells, timeline cells, item picker modal, Manager cost dashboard, and OM exchange rate utility.
- Stabilized User A `Add / Reuse Item` tables:
  - short `Add` action button
  - fixed action/detail columns
  - clamped item/spec/source text
  - scrollable modal body and sticky footer
- Stabilized Manager B `Demand Cost Dashboard`:
  - Excel-like table remains high density
  - amount/unit/total columns use fixed readable widths
  - full values remain available through titles/details
- Stabilized OM `Exchange Rate`:
  - compact inline utility
  - small `Save Rate` button
  - no large workflow panel behavior
- Added `tests/layout-smoke.js` for Playwright-based visual layout checks when Playwright is available.
- Updated `test.sh` to syntax-check and optionally run layout smoke.
- Updated system contract tests to guard the v1.3 layout contract.

## Test Result

- `./test.sh`: pass

- Browser smoke: skipped because Playwright is not installed

## Remaining Risk

The local environment still cannot run DOM layout measurements because Playwright is unavailable, and the Codex app window cannot be inspected by computer-use for safety reasons. The new layout smoke test will catch overlap and page overflow once Playwright is installed.

## UI Contract Going Forward

- Do not add long table button text. Use `Add`, `Edit`, `Remove`, `Detail`, `Export`, `Send`.
- Do not put full spec, PO, material number, purpose, remark, or history text in dense table cells.
- Every wide or dense table must use a `.table-shell`.
- Every action column must use `.cell-action` or an equivalent fixed-width action contract.
- Manager Excel-like tables may be wide, but page-level horizontal overflow is not allowed.
