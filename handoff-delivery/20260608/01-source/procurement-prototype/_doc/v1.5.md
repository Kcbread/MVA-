# v1.5 Global Text Visibility Contract

## Summary

This version fixes the root cause behind repeated clipped text issues. The UI must classify text before styling it. Identity text must remain recognizable; spec text may be summarized but must remain readable and traceable.

## Text Types

- **Identity**: item name, factory material no, request id, PAS demand no, PAS material no, package code, PO no, contact name/email. Must show enough information to identify the row.
- **Spec**: product specification. May show a 2-3 line summary in the main table, but full text must be available through Detail, tooltip, or row expand.
- **Note**: purpose, remark, pending reason, source explanation. Short summary in the table; full text in Detail.
- **Number**: qty, price, amount, percent. Do not wrap; use compact format where needed.
- **Action**: short button text only.

## Rules

- Do not clamp identity text with generic `cell-text-clamp`.
- Do not apply `display: -webkit-box` to `td`; clamp only an inner wrapper.
- Tables must scroll inside `.table-shell`, not at page level.
- If any visible text is truncated, the full value must be accessible.

## Regression Tests

- `tests/text-visibility-contract.test.js` validates the CSS classes and overrides.
- Existing layout smoke can be used to verify visible overlap when Playwright is available.

## Examples

- `Mini PC (Assy)` + `FM-VN-MVA-6D0F200` is identity text and must show two readable lines.
- `LAMITOUCH LM-30 CPU:i3-6100, RAM 16GB...` is spec text; the table can summarize it, but the full text must be available in Detail or tooltip.
