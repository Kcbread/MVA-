# v0.7 OPM Request Workspace Consolidation

## Completed

- Consolidated OPM/User A navigation to:
  - `Request Workspace`
  - `Action Required`
  - `Request Status`
- Removed `My Demand Overview` and `Reuse Item` from the primary tab row.
- Added `Add / Reuse Item` popup as the single item entry workbench.
- Merged item entry modes into the popup:
  - Search Catalog
  - Reuse Approved History
  - Reuse Project Package
  - Create New Item
- Simplified `Draft Items` into a dense short-summary table:
  - Select
  - Item
  - Spec Summary
  - Qty / Phases
  - Demand Rows
  - Breakdown Status
  - Status
  - Timeline
  - Actions
  - Detail
- Removed long `Purpose`, source text, and redundant project type columns from the Draft main table.
- Merged previous `My Demand Overview` behavior into `Request Status` by showing demand-ready draft rows and submitted rows in one tracking table.
- Kept the existing timeline-chip style for request tracking.
- Updated bilingual IT handoff docs and project decisions.
- Updated system contract tests to prevent the old split tabs from returning.

## UX Rationale

The requester page should answer three questions:

1. What do I need to add?
2. Where do I enter quantity?
3. Has this request moved forward?

The previous layout made users scan separate pages for Add Item, Reuse Item, My Demand Overview, and Request Status. The new layout keeps item acquisition in one popup and keeps package tracking in one dense status table.

## Remaining Risks

- The popup reuses existing catalog/history tables, so table density is improved but not fully redesigned.
- `app.js` is still oversized and should continue moving toward module ownership.
- Browser smoke remains skipped unless Playwright is installed locally.
