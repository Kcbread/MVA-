## UI Review Baseline

Every UI change must answer:

- Can users see the main point?
- Can users understand it?
- Can users take the correct action?
- Is the visual language consistent?

## Manager B Exception

Manager B `Cost Dashboard` and `Station Matrix` are intentional Excel-like high-density views. They should remain dense, but must keep readable headers, stable columns, and clear drilldown.

## Remaining Risks

- `app.js` and `real-data-seeds.js` are still oversized and should continue to be split into modules.
- Accessibility smoke depends on local Playwright and axe-core availability.
- Flesch-Kincaid is only a reference for English text; Chinese / mixed-language UI still needs manual review.
