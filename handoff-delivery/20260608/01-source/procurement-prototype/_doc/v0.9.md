# v0.9 Layout Stabilization

## 中文摘要

本版專注修正重複出現的版面穩定性問題，而不是新增新流程：

- User A `Add / Reuse Item` modal 改成 compact workspace，Add action 固定可見，不再需要水平拉到底。
- Reuse tables 只保留主畫面需要的短欄位；長 spec、PO、material no 進 Detail 或 tooltip。
- Manager B `Demand Cost Dashboard` 保留 Excel Dashboard-like 高密度，但金額改用 compact display，完整金額放 title/detail，避免 `$117,15...` 這種失去語意的截斷。
- OM `Exchange Rate` 改成小型 inline editor：Month / USD to VND / Save Rate / Current Rate Status。
- Shared dense table rule 重新整理：文字欄 clamp，數字欄 nowrap，主表不塞長段落。

## English Summary

This version focuses on layout stabilization rather than new workflow scope:

- User A `Add / Reuse Item` becomes a compact workspace with visible row actions; users no longer need horizontal scrolling to add items.
- Reuse tables keep only concise main-table columns. Long spec, PO, and material values move to Detail or tooltip.
- Manager B `Demand Cost Dashboard` keeps the Excel Dashboard-like dense style, but amount cells use compact readable labels while full amounts remain available in title/detail.
- OM `Exchange Rate` becomes a compact inline editor: Month / USD to VND / Save Rate / Current Rate Status.
- Shared dense table rules are tightened: clamped text columns, nowrap numeric columns, and no long prose in primary tables.

## Acceptance Notes

- User A item picker row actions must be visible without horizontal scroll.
- Manager cost dashboard must preserve Excel Dashboard headers and readable amount cells.
- OM exchange-rate save action must be compact and role-aware.
- `./test.sh` is the standard regression entrypoint.

## Remaining Risk

- This version stabilizes layout within the current classic-script prototype. Full modular extraction is still required because `app.js` remains too large for maintainable IT implementation.
- Browser smoke may be skipped when local Playwright is not installed, but the skip must be explicit in `./test.sh`.
