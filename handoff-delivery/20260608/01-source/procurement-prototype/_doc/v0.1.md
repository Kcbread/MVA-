# v0.1 Baseline - OPM / Manager B / OM Purchasing

## Scope
- Prototype scope covers OPM / User A, Manager B, OM Purchasing, Contacts, and Admin support views.
- Main workflow: OPM submits demand, Manager approves, OM records PAS / quote / export data.
- Quantity Matrix uses submitted demand rows immediately; it does not wait for OM or Buyer completion.

## Known Baseline
- OPM creates item demand and submits to Manager B.
- Manager B reviews approval queue and checks demand through progress and quantity views.
- OM Purchasing handles PAS Demand No, PAS quote result, User A confirmation, and export package.

## Risks Before v0.2
- `app.js` and `real-data-seeds.js` exceed 1000 lines and need module extraction.
- Some legacy override / enhancement code existed after the main render flow.
- Temporary budget and quote validity UI had previously been injected outside its owning module.

## Test Baseline
- Static syntax checks existed manually through `node --check`.
- No single project-wide `test.sh` existed.
- No formal unit or system contract tests existed.
