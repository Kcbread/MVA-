# v1.4 Row Height Contract

## Summary

This version closes the missing part of the shared layout contract: table row height.

Previous layout work controlled width, overflow, table shells, and button text, but table height was still content-driven. In HTML tables, row height expands whenever a cell contains long item names, specs, nested reason text, timeline chips, or stacked actions. That is why rows could still look broken even after width and overflow rules were added.

## Root Cause

- Row height was not part of the shared UI contract.
- Table cells inherited different padding and line-height from older CSS sections.
- Long `Item / Spec / Purpose / reason-text` content could expand rows instead of being clamped.
- Tests checked class names, but did not require row-height tokens or clamp rules.

## New Contract

All major table types must use shared row-height and vertical rhythm tokens:

- `--table-row-compact`: catalog, reuse, and dense search rows.
- `--table-row-standard`: regular dashboard rows.
- `--table-row-workflow`: workflow rows with status, timeline, and action buttons.
- `--table-cell-py` / `--table-cell-px`: controlled cell padding.
- `--table-line-height`: stable line height.
- `--table-clamp-2` / `--table-clamp-3`: text and spec clamp heights.

## Table Behavior

- Dense search/reuse tables use compact rows.
- Draft/request workflow tables use workflow rows.
- Long text is clamped in the main table.
- Full text belongs in Detail, tooltip, or modal.
- Reason text is single-line ellipsis inside dense/workflow tables.
- Status chips and table buttons are reduced to compact height.

## Tests

Added:

- `tests/row-height-contract.test.js`

This test validates that row-height tokens, vertical padding, line-height, and clamp rules exist in `styles.css`.

## Remaining Risk

Automated visual overlap detection still depends on Playwright. If Playwright is not installed, browser layout smoke is skipped and must be manually checked in the browser.
