# v1.2 Shared Table & Button Layout Standard

## Scope

This version establishes a reusable layout contract for procurement prototype tables and table actions.

## What Changed

- Added shared CSS utilities:
  - `.table-shell`
  - `.table-fixed`
  - `.form-table`
  - `.dense-dashboard-table`
  - `.workflow-table`
  - `.matrix-table`
  - `.cell-text`
  - `.cell-spec`
  - `.cell-number`
  - `.cell-action`
  - `.cell-timeline`
- Applied the standard to:
  - User A Draft Items
  - Add / Reuse Item catalog table
  - Reuse by Item table
  - Reuse by Project Package table
  - Manager Demand Cost Dashboard
- Changed table action labels to short verbs where needed, such as `Edit`.
- Added system contract tests for table shell, cell classes, and long-button prevention.

## Business Reason

The prototype had repeated layout regressions because global table styles, old overrides, long button labels, and timeline chips were mixed without a shared contract. v1.2 turns layout expectations into reusable CSS and testable contracts.

## Acceptance

- Table buttons must not overflow or cover neighboring cells.
- Long spec text is summarized in the table and available via tooltip or Detail.
- Dense Manager tables may remain Excel-like, but must preserve readable numeric cells.
- Any future table must declare its table type and cell roles.

## Test Notes

- Run `./test.sh`.
- Browser and accessibility smoke may be skipped only when Playwright / axe-core are unavailable, and the skip must be reported.
